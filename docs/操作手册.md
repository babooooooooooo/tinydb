# tinydb 操作手册

tinydb 是一个用纯 Python 实现的轻量级嵌入式关系数据库：磁盘分页存储、
B+ 树索引、Volcano 风格执行器、单写者 WAL 保证崩溃安全，运行时零依赖。

本手册面向使用者，涵盖安装、启动、命令行 / Python API 用法、SQL 参考、
事务、崩溃恢复与备份、故障排查以及能力边界。

---

## 1. 安装

需要 Python 3.10+（代码使用 `X | Y` 类型语法与 `slots` dataclass）。

```bash
# 在项目根目录，建议使用虚拟环境
python3 -m venv .venv
source .venv/bin/activate
pip install -e .[dev]
```

`-e .[dev]` 会安装 tinydb 本身以及测试依赖（pytest 等）。

> 注意：本机 `python` 命令可能不存在，请统一使用 `python3`。若已激活项目自带的
> `.venv`，则 `python` 通常已可用。

## 2. 数据文件说明

一个数据库对应两个文件，二者是一个整体：

| 文件 | 作用 |
| --- | --- |
| `<name>.db` | 主数据文件（分页存储：表头、堆页、B+ 树、目录页） |
| `<name>.db.wal` | 预写日志（Write-Ahead Log），用于崩溃恢复 |

打开数据库时若文件不存在会自动创建；若 `.wal` 中存在上次会话未落盘的记录，
会在打开时自动重放（recover）。**备份或迁移时必须同时复制这两个文件。**

## 3. 命令行 REPL

### 启动

```bash
python -m tinydb example.db
```

进入交互式提示符：

```
tinydb v0.1.0 — file: example.db
Type .help for meta-commands or .exit to quit.
tinydb>
```

### 输入规则

- SQL 语句以分号 `;` 结束；未遇到分号时提示符变为 `     -> `，支持多行输入。
- 顶层空行、以及只有 `;` 的行会被忽略。
- 字符串字面量中的分号不会被误判为语句结束。

### 元命令（Meta-commands）

元命令以点号 `.` 开头，不是 SQL，由 REPL 直接解释：

| 命令 | 说明 |
| --- | --- |
| `.tables` | 列出所有表 |
| `.schema <表名>` | 打印该表的 `CREATE TABLE` 语句及索引 |
| `.help` | 显示所有元命令 |
| `.exit` / `.quit` | 退出 REPL |

### 示例会话

```
tinydb> CREATE TABLE users (id INT PRIMARY KEY, name TEXT NOT NULL);
tinydb> INSERT INTO users VALUES (1, 'alice');
1 row affected
tinydb> SELECT * FROM users;
id | name
---+------
1  | alice
tinydb> .schema users
CREATE TABLE users (
    id INT PRIMARY KEY,
    name TEXT NOT NULL
);
tinydb> .exit
```

## 4. Python API

### 基本用法

```python
from tinydb import Database

with Database("example.db") as db:            # 退出 with 块时自动 close()
    db.execute("CREATE TABLE users (id INT PRIMARY KEY, name TEXT NOT NULL)")
    db.execute("INSERT INTO users VALUES (1, 'alice')")
    rs = db.execute("SELECT * FROM users ORDER BY id")
    print(rs.columns)        # ['id', 'name']
    for row in rs.rows:
        print(row)           # ['1', 'alice']  —— 所有字段均为字符串
```

### 主要方法

| 方法 | 说明 |
| --- | --- |
| `Database(path, *, buffer_capacity=64)` | 打开 / 创建数据库；`buffer_capacity` 为缓冲池页数 |
| `execute(sql) -> ResultSet` | 执行一条或多条（`;` 分隔）语句，返回**最后一条**的结果 |
| `executemany(sql) -> list[ResultSet]` | 执行脚本并返回**每一条**语句的结果 |
| `begin()` / `commit()` / `rollback()` | 显式事务控制（等价于 SQL 的 `BEGIN/COMMIT/ROLLBACK`） |
| `close()` | 落盘目录 + 刷新脏页并关闭文件；可重复调用 |
| `closed` | 属性，是否已关闭 |

### ResultSet 结构

```python
@dataclass
class ResultSet:
    columns: list[str]        # 投影列名；写操作为空
    rows: list[list[str]]     # 每行是字符串列表（每列一个），NULL 渲染为 ""
    rows_affected: int        # 仅写操作有意义
```

> 说明：结果中的所有值都是**字符串**；NULL 显示为空字符串 `""`。数值需要自行
> `int(...)` / `float(...)` 转换。

## 5. SQL 参考

### 5.1 数据类型

| 类型 | 说明 |
| --- | --- |
| `INT` | 64 位整数（`SUM` 在整数域累加，不会丢精度） |
| `SMALLINT` | 16 位整数，范围 -32768..32767 |
| `BIGINT` | 64 位整数，范围 ±2⁶³ |
| `FLOAT` | 双精度浮点；拒绝 `inf` / `NaN` |
| `DOUBLE` / `REAL` | `FLOAT` 的别名 |
| `TEXT` | 字符串 |
| `VARCHAR(N)` | 长度上限 N 的字符串；超过 N 写入被拒 |
| `CHAR(N)` | 固定长度 N 的字符串；右补空格；超过 N 写入被拒 |
| `BOOL` | 布尔（`TRUE` / `FALSE`） |
| `BOOLEAN` | `BOOL` 的别名 |
| `DATE` | 日期（字面量 ISO-8601：`'2025-01-15'`） |
| `TIME` | 时间（字面量 ISO-8601：`'13:45:30'`） |
| `TIMESTAMP` | 时间戳（字面量 ISO-8601：`'2025-01-15T13:45:30'`，可加 `Z` 表示 UTC） |
| `DECIMAL(p, s)` | 精确数值，以规范化字符串存盘；总位数 ≤ p、小数位 ≤ s |

### 5.2 约束

| 约束 | 说明 |
| --- | --- |
| `PRIMARY KEY` | 主键；每表至多一个；自动建唯一索引 |
| `NOT NULL` | 非空 |
| `UNIQUE` | 唯一；自动建唯一索引 |

`PRIMARY KEY` 与 `UNIQUE` 列会自动创建索引（索引根页在首次插入时惰性分配），
以便快速做唯一性校验和等值 / 范围查找。

### 5.3 DDL

```sql
CREATE TABLE t (
    id    INT PRIMARY KEY,
    email TEXT UNIQUE,
    name  TEXT NOT NULL,
    age   INT
);

DROP TABLE t;
```

### 5.4 INSERT

```sql
-- 提供全部列（按定义顺序）
INSERT INTO users VALUES (1, 'alice');

-- 指定列子集，未列出的列填 NULL（须满足 NOT NULL / 主键约束）
INSERT INTO users (id, name) VALUES (2, 'bob');
```

### 5.5 SELECT

支持的子句顺序：

```sql
SELECT [DISTINCT] <列 | * | 聚合(...)> [AS 别名], ...
FROM <表> [别名]
[WHERE <条件>]
[GROUP BY <列>, ...]
[ORDER BY <表达式> [ASC|DESC], ...]
[LIMIT <n>]
[OFFSET <n>]
```

示例：

```sql
SELECT * FROM orders WHERE amount > 100 ORDER BY amount DESC LIMIT 10;
SELECT DISTINCT region FROM orders;
SELECT region, COUNT(*), SUM(amount) FROM orders GROUP BY region ORDER BY region;
SELECT id AS order_id, amount FROM orders OFFSET 5;
```

### 5.6 聚合函数

| 函数 | 语义 |
| --- | --- |
| `COUNT(*)` | 行数 |
| `COUNT(col)` | 非 NULL 值个数 |
| `SUM(col)` | 非 NULL 之和；全为 NULL / 无输入时返回 NULL（不是 0） |
| `AVG(col)` | 非 NULL 的平均；无输入返回 NULL |

**不支持 `MIN` / `MAX`**（解析阶段会报错 `unknown function`）。
标量聚合（无 `GROUP BY`）在空表上仍返回 1 行；带 `GROUP BY` 时空输入返回 0 行。

### 5.7 UPDATE / DELETE

```sql
UPDATE users SET name = 'ALICE', age = 30 WHERE id = 1;
DELETE FROM users WHERE age < 18;
```

省略 `WHERE` 会作用于全表。修改被索引的列时，索引会同步维护；若索引维护失败，
写操作会**报错中止**（不会静默留下损坏的索引）。

### 5.8 表达式与运算符

- 字面量：整数、浮点、字符串（单引号，内部 `''` 表示一个 `'`）、`TRUE`/`FALSE`、`NULL`。
- 列引用：`col` 或 `表.col`。
- 算术：`+` `-` `*` `/`，一元负号 `-x`，括号 `( )`。
- 比较：`=`、`<>`、`<`、`>`、`<=`、`>=`。
  **不等号只支持 `<>`，不支持 `!=`。**
- 逻辑：`AND`、`OR`、`NOT`。
- NULL 采用 SQL 三值逻辑（`TRUE` / `FALSE` / `UNKNOWN`）：任何与 NULL 的
  比较结果为 UNKNOWN，`WHERE` 中 UNKNOWN 视为不满足。

## 6. 事务

tinydb 是**单写者**模型：同一时刻只允许一个事务，嵌套 `BEGIN` 会报错。

- **自动提交**：未显式 `BEGIN` 时，每条写语句作为独立事务自动提交。
- **显式事务**：

  ```sql
  BEGIN;
  UPDATE accounts SET balance = balance - 100 WHERE id = 1;
  UPDATE accounts SET balance = balance + 100 WHERE id = 2;
  COMMIT;      -- 或 ROLLBACK 撤销本事务的全部改动
  ```

  Python API 对应 `db.begin()` / `db.commit()` / `db.rollback()`。

- **关闭时**：若仍有未提交事务，`close()` 会执行隐式回滚。

## 7. 崩溃恢复与备份

### 恢复

WAL 采用整页日志 + LSN。提交时先把脏页写入 WAL 再写 `COMMIT` 记录并 fsync。
下次 `Database(path)` 打开时自动重放 WAL：已提交事务的记录会被应用，
没有匹配 `COMMIT` 的记录会被丢弃。使用者无需手动干预。

### 备份

在数据库**已关闭**（进程退出或 `close()` 之后）时，同时复制：

```bash
cp example.db      backup/example.db
cp example.db.wal  backup/example.db.wal
```

恢复时把两个文件放回原路径再打开即可。**切勿只复制 `.db` 而遗漏 `.wal`**，
否则可能丢失最近一次会话尚未合并的改动。

## 8. 能力边界（当前版本 v0.1.0）

- 仅支持**单表**查询，**没有 `JOIN`**、子查询、视图。
- 聚合仅 `COUNT` / `SUM` / `AVG`，**无 `MIN` / `MAX`**、无 `HAVING`（写了会被忽略）。
- 无 `ALTER TABLE`、无 `CREATE INDEX`（索引仅由 `PRIMARY KEY` / `UNIQUE` 自动创建）。
- 不等号仅 `<>`。
- 单进程、单写者，无并发 / 无网络协议。

## 9. 故障排查

| 现象 | 可能原因 / 处理 |
| --- | --- |
| `error: no such table: X` | 表不存在，先 `CREATE TABLE` 或检查表名大小写 |
| `error: unknown function 'MIN'` | 当前版本不支持 MIN/MAX，改用受支持的聚合或应用层计算 |
| `error: expected FROM` 等解析错误 | 检查 SQL 语法；注意不等号用 `<>`、字符串用单引号 |
| 唯一 / 主键冲突报 `ConstraintError` | 插入 / 更新违反 `PRIMARY KEY` 或 `UNIQUE` |
| REPL 一直显示 `-> ` 不返回 | 语句未以 `;` 结束，或字符串引号未闭合 |
| 打开后数据比预期旧 | 确认备份时同时保留了 `.db` 与 `.db.wal` |
