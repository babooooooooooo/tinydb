## 3. 测试方法论

### 3.1 单元测试（tests/unit/，550 用例）

- 隔离每个模块的内部接口
- 大量使用 `pytest.mark.parametrize` 覆盖等价类边界
- 临时文件 / 内存对象作 fixture（不依赖外部状态）
- 目标覆盖率：每个模块 ≥ 85%

### 3.2 集成测试（tests/integration/，78 用例）

按子套件组织：

| 子套件 | 文件 | 用例数 | 方法 |
| --- | --- | --- | --- |
| 端到端 SQL workflow | `test_end_to_end.py` | 11 | 真磁盘 DB；CREATE / INSERT / UPDATE / DELETE / SELECT 全链路；含 close-reopen 持久化 |
| 事务 ACID | `test_transactions.py` | 32 | BEGIN / COMMIT / ROLLBACK 的 4 个隔离性质、并发不变量、WAL 恢复 |
| 差分 / 模糊 | `test_differential_fuzz.py` | 5 | 随机 SQL 序列 vs Python 内存参考模型；持久化测试用 record-and-replay 模式 |
| 新类型 E2E | `test_new_types.py` | 22 | SQL92 8 个新标量 + DOUBLE/REAL/BOOLEAN 别名的端到端 round-trip + 边界拒绝 + 接受 |
| CLI 子进程 | `test_cli.py` | 8 | `python -m tinydb <file>` 启动 + REPL 交互 |

### 3.3 关键设计原则

1. **无 mock**：所有集成测试都启动真实的 `Database` 实例，落盘到 `tmp_path`。
2. **TDD 流程**：所有新功能 / bug 修复都遵循 RED（失败用例）→ GREEN（最小实现）→ VERIFY（完整套件 + 覆盖率）三步走。
3. **差分测试**：随机 SQL 序列与一个 Python 内存参考模型同步推进，任何不一致都是 bug。
4. **持久化 record-and-replay**：先从种子 RNG 记录**操作列表**（与状态无关），再分别重放——避免重放 RNG 导致 INSERT 在已存在行上重放而掩盖持久化 bug。

## 7. 已知限制

1. **不提供 VERSION=1 → VERSION=2 迁移**：C4 commit 决定。若有旧 `.db` 文件，启动时
   会以 `StorageError: unsupported version: 1` 干净拒绝。解决方案：删除旧文件
   重灌，或在 C4 之前的 commit 导出数据。
2. **REPL 启动时若 DB 损坏直接 traceback 退出**：未在 `run_repl` 中捕获 `StorageError`。
   UX 不够友好，可作为后续改进。
3. **持久化测试只覆盖最常见的 catalog-flush 类 bug**：对 heap 页 corruption 的检测
   能力有限（DB 自身对损坏 heap 页有一定的容错解析）。
4. **集成测试不覆盖 64-bit 文件大小极限、磁盘满、并发 fsync 冲突等极端场景**。

## 8. 如何运行

### 8.1 完整套件

```bash
source .venv/bin/activate
python -m pytest
```

### 8.2 集成测试子集

```bash
# 全部集成测试
python -m pytest tests/integration -q

# 端到端
python -m pytest tests/integration/test_end_to_end.py -v

# 事务 ACID
python -m pytest tests/integration/test_transactions.py -v

# 差分 / 模糊
python -m pytest tests/integration/test_differential_fuzz.py -v

# 新类型 E2E
python -m pytest tests/integration/test_new_types.py -v
```

### 8.3 覆盖率报告

```bash
python -m pytest --cov=tinydb --cov-report=term-missing
```

`--cov-report=html` 生成 `htmlcov/index.html` 浏览器可查看。

### 8.4 端到端手工 smoke

```python
from tinydb.database import Database

with Database(":memory:") as db:                       # in-memory
    db.execute("CREATE TABLE t (id BIGINT PRIMARY KEY, name VARCHAR(20), price DECIMAL(10,2))")
    db.execute("INSERT INTO t VALUES (1, 'alice', '3.14')")
    print(db.execute("SELECT * FROM t").rows)
# 期望: [['1', 'alice', '3.14']]
```
